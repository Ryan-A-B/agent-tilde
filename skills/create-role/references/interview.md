# Create Role Interview

Use this interview before running `scripts/create-role.sh`.

The goal is to create roles that are broad, stable, and non-overlapping.

For obvious broad roles, do not start with this full interview. Instead:

1. draft the role directly
2. show the draft to the human
3. refine it if needed
4. scaffold it after approval

Use the fuller interview only when the requested role is ambiguous, narrow, overlapping, or likely to be better modeled as a skill.

## Required Questions

1. What should this role be called?
2. What core work should this role own?
3. What is explicitly out of scope for this role?
4. How is this role different from the existing roles?

## Challenge Questions

1. Is this actually a new role, or would a skill on an existing role be enough?
2. Will this role stay useful across many projects, or is it too project-specific?
3. Does this role overlap too much with `developer`, `marketer`, `qa`, `support`, or another existing role?

## Operator Notes

- Push toward broad, durable roles.
- Prefer specialization through skills and project memory rather than through narrowly sliced roles.
- If the proposed role overlaps heavily with an existing role, say so and suggest the simpler alternative.
- Do not scaffold immediately when the draft still needs work; iterate in chat first.
- Use `references/examples/` when a broad example role would help seed the first draft.
