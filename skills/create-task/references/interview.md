# Create Task Interview

Use this interview before running `scripts/create-task.sh`.

The goal is to create a task that is clear enough to be resumed or handed off later without re-explaining the work.

## Required Questions

1. Which area does this task belong to?
2. What should the task be called?
3. What is the goal?
4. What description should be captured in the task?
5. Who should own the task first?

## Optional Questions

1. Are any repositories especially relevant?
2. Is there a preferred priority?
3. Is this task intended as immediate work or a backlog item?

## Operator Notes

- Do not guess the area.
- Default assignment should usually be an agent role, not `human`.
- If the correct assignee is unclear, ask before scaffolding.
